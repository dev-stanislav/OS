# MiniOS

> Экспериментальное 32-битное x86-ядро, написанное на C и NASM.

MiniOS — не готовая операционная система и не замена Linux или Windows. Это эксперимент: может ли ИИ в 2026 году помочь человеку создать собственное работающее ядро с нуля. Ядро загружается в QEMU, рисует текстовый интерфейс и имеет собственную консоль.

## Что уже работает

- загрузка ELF-ядра в QEMU;
- GDT и заготовка IDT;
- текстовый VGA-режим 80×25;
- polling-ввод с PS/2-клавиатуры;
- shell-команды `help`, `clear`, `about`, `game`;
- мини-игра «Угадай число».

## Запуск

Проект рассчитан на **MSYS2 UCRT64** в Windows. Нужны `make`, `nasm`, `clang`, `ld.lld` и `qemu-system-i386`.

Проверь, что инструменты доступны:

```sh
make --version
nasm --version
clang --version
qemu-system-i386 --version
```

Из корня папки `kernel` запусти:

```sh
make clean
make run
```

QEMU откроет отдельное окно с MiniOS. Для выхода нажми `Ctrl+A`, затем `X`.

## Команды MiniOS

| Команда | Что делает |
| --- | --- |
| `help` | Показывает список команд |
| `clear` | Очищает экран |
| `about` | Кратко описывает систему |
| `game` | Запускает игру «Угадай число» |

В игре введи цифру от `1` до `9`. Команда `exit` возвращает в shell.

## Может ли ИИ написать собственное ядро в 2026 году?

Да, но не в одиночку и не без проверки. ИИ может писать заготовки драйверов, объяснять x86, находить ошибки линкера и помогать с C и ассемблером. Человек задаёт направление, запускает код, проверяет результат и принимает технические решения.

Ядро нельзя просто скопировать вслепую: оно работает с памятью, прерываниями и портами устройств, где одна неточность легко превращается в чёрный экран. Поэтому MiniOS проверяет практическую границу возможностей: насколько далеко можно зайти, создавая ОС в диалоге с ИИ и проверяя каждый шаг в QEMU.
