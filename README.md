# MiniOS

> Экспериментальное 32-битное x86-ядро, написанное на C и NASM.

MiniOS — не готовая операционная система и не замена Linux или Windows. Это эксперимент: может ли ИИ в 2026 году помочь человеку создать собственное работающее ядро с нуля. Ядро загружается в QEMU, рисует текстовый интерфейс и имеет собственную консоль.

## Что уже работает

- загрузка ELF-ядра в QEMU;
- GDT, IDT, PIC и аппаратные IRQ таймера/клавиатуры;
- текстовый VGA-режим 80×25;
- аппаратный курсор, история команд и редактирование строки;
- PS/2-клавиатура: Shift, Caps Lock, Backspace, стрелки, Home/End и Delete;
- простая файловая система на persistent QEMU-диске с папками и файлами;
- kernel bump-heap на 128 KiB и identity-paging первых 16 MiB;
- RTL8139, Ethernet, ARP и ICMP ping через QEMU user-network;
- shell-команды для навигации, файлов и системной информации;
- мини-игра «Угадай число».

## Запуск

Проект рассчитан на **MSYS2 UCRT64** в Windows. Нужны `make`, `nasm`, `clang`, `ld.lld` и `qemu-system-i386`.

Проверь, что инструменты доступны:

```sh
make --version
nasm --version
clang --version
qemu-system-i386 --version
```

Из корня репозитория запусти:

```sh
cd kernel
make clean
make run
```

QEMU откроет отдельное окно с MiniOS. Для выхода нажми `Ctrl+A`, затем `X`.

### Быстрый запуск с нуля

1. Установи [MSYS2](https://www.msys2.org/) и открой терминал **MSYS2 UCRT64**. Не PowerShell и не обычный MSYS — именно UCRT64.
2. Установи инструменты один раз:

   ```sh
   pacman -S --needed make nasm mingw-w64-ucrt-x86_64-clang mingw-w64-ucrt-x86_64-lld mingw-w64-ucrt-x86_64-qemu
   ```

3. Перейди в папку с проектом и запусти систему:

   ```sh
   cd /c/Users/ТВОЁ_ИМЯ/Documents/OS/kernel
   make run
   ```

4. В окне QEMU появится приглашение `minios:/ >`. Для первой проверки введи:

   ```text
   help
   minipkg list
   minipkg install minifetch
   run minifetch
   net ping
   net info
   ```

### Диск и сброс данных

- При первом `make run` автоматически создаётся `minios.img` — диск MiniOS.
- `make clean` пересобирает ядро, но сохраняет файлы и установленные пакеты на диске.
- Если нужен совершенно чистый MiniOS, выполни `make distclean`, затем `make run`.

### Если не запускается

- `qemu-system-i386 not found` — установи QEMU командой из шага 2 и перезапусти UCRT64.
- `clang` или `nasm not found` — установи недостающий пакет из шага 2.
- Чёрный экран после изменения ядра — выполни `make clean && make run` и проверь последние изменения.

## MiniPkg

`minipkg` — локальный менеджер приложений. В первой версии пакеты встроены в ядро, а установка сохраняется только в RAM до перезагрузки.

```text
minipkg list
minipkg install minifetch
minipkg install calc
minipkg installed
run minifetch
run calc 12 * 7
minipkg remove calc
```

Каталог исходных manifest-файлов лежит в [`packages`](packages). Доступны `minifetch`, `calc`, `clock`, `files`, `matrix`, `mines` и `free`. Для интерактивных приложений нажми `Esc` или `Ctrl+C`, чтобы вернуться в shell.

## Редактор Free

`free` — встроенный полноэкранный текстовый редактор для файлов на диске.

```text
minipkg install free
run free notes.txt
```

Стрелки перемещают курсор, `Backspace` и `Delete` удаляют символы, `Enter` создаёт новую строку. Нажми `Ctrl+S`, чтобы сохранить файл, и `Ctrl+C` или `Esc`, чтобы выйти.

## TBF — The Best File

`tbf` — встроенный консольный файловый менеджер, его не нужно устанавливать через MiniPkg.

```text
tbf
```

Стрелки выбирают файл или папку, `Enter` открывает папку, `Space` показывает содержимое файла, а `Backspace` возвращает на уровень выше. `Esc` или `Ctrl+C` возвращают в shell.

## Сеть

`make run` автоматически запускает QEMU с RTL8139 и user-network. Это первый слой сети: адаптер получает фиксированный адрес QEMU user-network `10.0.2.15`, а шлюз — `10.0.2.2`.

```text
net info
net ping
net info
net ping 8.8.8.8
```

`net ping` отправляет ARP-запрос и ICMP echo на шлюз. DNS, TCP, HTTP и скачивание пакетов пока не реализованы — они будут следующими слоями перед сетевым MiniPkg.

## Процессы и user mode

MiniOS содержит первый ring-3 процесс и системные вызовы через `int 0x80`. Проверь переход из kernel mode в user mode и обратно:

```text
ps
userdemo
ps
```

`userdemo` пишет строку через syscall `write`, затем завершает работу через syscall `exit`. Это первый шаг к процессам: scheduler и запуск отдельных ELF-программ ещё впереди.

В игре введи цифру от `1` до `9`. Команда `exit` возвращает в shell.

Пример работы с файлами:

```text
mkdir docs
cd docs
write note "hello world"
cat note
cd /
ls
```

Файлы, а также установленные MiniPkg-приложения, сохраняются в `kernel/minios.img` и переживают перезапуск QEMU. `make clean` не удаляет диск; для полного сброса используй `make distclean`.

## Может ли ИИ написать собственное ядро в 2026 году?

Да, но не в одиночку и не без проверки. ИИ может писать заготовки драйверов, объяснять x86, находить ошибки линкера и помогать с C и ассемблером. Человек задаёт направление, запускает код, проверяет результат и принимает технические решения.

Ядро нельзя просто скопировать вслепую: оно работает с памятью, прерываниями и портами устройств, где одна неточность легко превращается в чёрный экран. Поэтому MiniOS проверяет практическую границу возможностей: насколько далеко можно зайти, создавая ОС в диалоге с ИИ и проверяя каждый шаг в QEMU.
